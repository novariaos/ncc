# ncc

Компилятор C для NVM на [NovariaOS](https://github.com/novariaos/novariaos-src/).

## Обзор

ncc компилирует подмножество C в NVM-ассемблер, затем собирает его в `.bin`-бинарники, которые выполняются на стековой виртуальной машине NVM.

**Пайплайн:** `C source -> препроцессор -> лексер -> парсер -> AST -> кодген -> NVM asm -> nvma -> .bin`

## Сборка

```
cargo build --release
```

## Использование

```
ncc [опции] <input.c>
```

| Опция | Описание |
|---|---|
| `-o <file>` | Выходной файл (по умолчанию: `input.bin`) |
| `--emit-asm` | Вывести сгенерированный NVM-ассемблер в stdout |
| `-I <dir>` | Добавить директорию для поиска заголовков |

Директория `include/` в текущей рабочей папке добавляется автоматически.

Также поддерживается прямая сборка `.asm` файлов:

```
ncc input.asm -o output.bin
```

## Пример

```c
#include "stdio.h"

int main() {
    printf("Hello, world!\n");
    return 0;
}
```

```
ncc examples/hello.c -o hello.bin
```

## Поддерживаемое подмножество C

### Типы
- `int`, `char`, `void`
- Указатели (`int *p`)
- Массивы (`int arr[10]`)
- Структуры (`struct Point { int x; int y; }`)

### Операторы (statements)
- `if` / `else`
- `while`, `do...while`, `for`
- `switch` / `case` / `default` / `break`
- `return`
- Объявления локальных переменных с инициализаторами

### Выражения
- Арифметика: `+`, `-`, `*`, `/`, `%`
- Сравнение: `==`, `!=`, `<`, `<=`, `>`, `>=`
- Логические: `&&`, `||`, `!`
- Присваивание: `=`, `+=`, `-=`, `*=`, `/=`, `%=`
- Инкремент/декремент: `++`, `--` (префиксный и постфиксный)
- Вызовы функций, индексация массивов, доступ к полям структур (`.`)
- `sizeof`, приведение типов
- Строковые и символьные литералы с escape-последовательностями

### Препроцессор
- `#include "file.h"` и `#include <file.h>`
- `#define NAME value`
- `#ifdef` / `#ifndef` / `#endif`
- Удаление `__attribute__((...))` из заголовков

### Заголовки стандартной библиотеки
- `stdio.h` — `printf` (intrinsic компилятора), `putchar`, `print_int`, `print_char`, `print_ln`
- `stdlib.h` — `abs`, `min`, `max`
- `string.h` — `strlen`, `strcmp`
- `nvm.h` — обёртки над вызывами NVM

## Целевая платформа NVM

- Стековая ВМ, 32-битные знаковые целые, big-endian
- 1024 ячейки стека, 512 локальных слотов процесса
- Вызовы через фреймы: `enter` / `leave` / `ret` / `load_arg`
- Вывод идёт через `/dev/tty` (открывается при старте программы)
- `printf` — intrinsic компилятора, формат-строка разбирается в compile time

## Структура проекта

```
src/
  main.rs          Точка входа CLI
  cc/              Фронтенд компилятора C
    token.rs       Типы токенов
    lexer.rs       Токенизатор
    preprocess.rs  Минимальный препроцессор C
    ast.rs         Типы узлов AST
    parser.rs      Рекурсивный парсер нисходящего разбора
    types.rs       Резолв типов и layout структур
    codegen.rs     Генерация NVM-ассемблера из AST
  nvm/             Определения NVM
    asm.rs         Билдер ассемблерного текста
    opcodes.rs     Константы опкодов
  ffi/             FFI к ассемблеру nvma
  nvma/            Ассемблер nvma на C (asm текст -> бинарник)
include/           Заголовки стандартной библиотеки C для NVM
```

## Ограничения

- Нет побитовых операций и сдвигов
- Нет арифметики указателей и разыменования (ограниченная поддержка указателей)
- Нет `enum`, `union`, `typedef`
- Доступ к массивам по переменному индексу через switch-dispatch (макс. ~64 элемента)
- Формат-строка `printf` должна быть строковым литералом
- `%s` в `printf` работает только со строковыми литералами
